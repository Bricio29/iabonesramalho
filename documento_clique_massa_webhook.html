Integração com ClickMassa via Webhook

Bot de Atendimento Inteligente — Bonés Ramalho

Projeto: Bot WhatsApp — Bonés Ramalho
Data: Março de 2026
Status: Em implementação
Este documento detalha todas as alterações técnicas realizadas no sistema de atendimento automático para viabilizar a integração com a plataforma ClickMassa através do protocolo Webhook, bem como as etapas ainda necessárias para que a solução entre em operação completa.
O que foi implementado
🔄
Migração de Arquitetura
De biblioteca local para integração via API
  • O sistema foi migrado de uma integração por biblioteca (que exigia QR Code manual) para uma arquitetura modern baseada em API + Webhook
  • O bot agora opera como um servidor web independente, pronto para receber eventos do ClickMassa em tempo real
  • Eliminada a necessidade de manter sessão local do WhatsApp Web ativa no servidor
📡
Endpoint de Webhook Implementado
Rota POST /webhook — servidor Express.js na porta 3000
  • Servidor HTTP criado para receber todos os eventos de mensagens enviados pelo ClickMassa
  • Resposta imediata com status 200 OK ao ClickMassa para evitar timeouts e reenvios duplicados
  • Processamento assíncrono das mensagens após a confirmação — garante performance e estabilidade
  • Rota de healthcheck (GET /health) para monitoramento do serviço
🔐
Autenticação e Segurança
Validação de token nas requisições recebidas
  • Implementada validação por token secreto nos headers de cada requisição recebida
  • Requisições sem token válido são rejeitadas com status 401 Unauthorized
  • Mensagens originadas do próprio bot são automaticamente ignoradas (sem loop de respostas)
  • Mensagens de grupos são descartadas — o bot atende apenas conversas individuais
📨
Suporte a Todos os Tipos de Mensagem
Texto, imagem, áudio e documento
  • Mensagens de texto — resposta inteligente via IA (OpenAI GPT)
  • Áudios / mensagens de voz — transcrição automática e resposta contextualizada
  • Imagens — recebimento e processamento via base64
  • Documentos — recebimento e identificação do tipo de arquivo
  • Compatibilidade com o formato Evolution API, padrão utilizado nas integrações ClickMassa
📤
Envio de Respostas via API do ClickMassa
Texto, imagens de catálogo e indicador de digitação
  • Função de envio de texto via endpoint /message/sendText/{instância}
  • Função de envio de imagens do catálogo em base64 via /message/sendMedia/{instância}
  • Indicador de "digitando..." antes de cada resposta via /chat/sendPresence/{instância}
  • Autenticação nas chamadas à API via header apikey: {token}
🧠
Inteligência Artificial Integrada
Mantida toda a lógica de atendimento existente
  • Toda a base de conhecimento sobre os bonés, modelos, preços e técnicas foi mantida
  • Histórico de conversa por cliente preservado em banco de dados local (database.json)
  • Envio automático de fotos do catálogo quando o cliente informa o objetivo de uso
  • Notificação automática para a equipe interna quando um lead qualificado é identificado

O que precisa ser feito para entrar em funcionamento
1
Configurar as variáveis de ambiente no servidor
Criar o arquivo .env no servidor com as credenciais do ClickMassa e da OpenAI. Todas as variáveis marcadas como obrigatórias precisam estar preenchidas corretamente para o bot funcionar.
Variável Descrição Tipo
CM_API_URL URL base da API do ClickMassa — ex: https://api.clickmassa.com.br Obrigatório
CM_TOKEN Token / API Key gerado no painel do ClickMassa Obrigatório
CM_INSTANCE Nome da instância/sessão do WhatsApp no ClickMassa Obrigatório
OPENAI_API_KEY Chave da API OpenAI para funcionamento da IA Obrigatório
EQUIPE_NUMERO Número do WhatsApp da equipe para notificações de leads (ex: 5584999999999) Obrigatório
WEBHOOK_SECRET Token secreto para validar que as requisições vêm do ClickMassa Recomendado
PORT Porta do servidor (padrão: 3000) Opcional

2
Subir o servidor em um ambiente com IP público
O ClickMassa precisa conseguir acessar o servidor para entregar os eventos de mensagem. O servidor deve estar online com endereço acessível pela internet — seja um IP público fixo, um domínio ou um túnel seguro (ex: ngrok para testes).
3
Configurar o Webhook no painel do ClickMassa
No painel de administração do ClickMassa, na instância correspondente, cadastrar a URL do webhook apontando para o servidor:

http://<IP_DO_SERVIDOR>:3000/webhook

Habilitar os eventos de mensagens recebidas (messages.upsert). Se o painel suportar header de segurança, configurar com o mesmo valor definido em WEBHOOK_SECRET.
4
Instalar dependências e iniciar o servidor
Executar npm install no diretório do projeto para instalar todas as dependências, depois iniciar com npm start ou node index.js. O servidor exibirá na inicialização a URL do webhook e alertas caso alguma credencial esteja faltando.
5
Validação e testes em produção
Enviar mensagens de teste para o número WhatsApp conectado à instância do ClickMassa e verificar se o bot responde corretamente. Confirmar o envio de fotos do catálogo, o funcionamento do indicador de "digitando" e a notificação de leads para a equipe interna.
ℹ️

Sobre o formato da API: A implementação já está preparada para o formato Evolution API, que é o mais comum nas integrações ClickMassa. Caso a instância utilize um formato de payload diferente, um ajuste pontual na função de leitura de mensagens será necessário — algo rápido de resolver com a documentação da instância em mãos.

⚠️

Ponto de atenção: As credenciais da API do ClickMassa (CM_API_URL, CM_TOKEN e CM_INSTANCE) são essenciais. Sem elas, o servidor receberá as mensagens normalmente, mas não conseguirá responder. É o único ponto de dependência externa que impede a operação completa no momento.
